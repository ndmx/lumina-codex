# Lumina Product Profiles

Product profiles adapt Lumina's canonical system to real apps.

Use product profiles for:

- Product-specific names and aliases.
- App-specific tab names, feature surfaces, privacy posture, and launch docs.
- Archived source snapshots that explain past implementation decisions.

Use `docs/system/` for:

- Canonical tags.
- Cross-platform naming rules.
- Reusable element/component models.
- Agent update workflow.

Current profiles:

- `mobile-design-systems/`: JxL Scheduler, ParkMemory Hub, shared iOS principles, and archived source docs.
